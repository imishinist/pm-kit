# プロジェクトセットアップ ヒアリング

## 目的

`pm-kit create` で生成されたプロジェクトの `project.yaml` を、ユーザーとの対話を通じて充実させる。

## 手順

以下の順序でユーザーにヒアリングし、回答を `project.yaml` に反映してください。

### 1. プロジェクト概要

- プロジェクトの目的・ゴールは何ですか？
- 一言で表すとどんなプロジェクトですか？（description に反映）

### 2. Jira 設定

- Jira を使っていますか？
- 使っている場合:
  - Jira の URL（例: `https://company.atlassian.net`）
  - プロジェクトキー（例: `RNW`）
  - ボード ID（ボードの URL から取得可能）
  - ボードタイプ: スクラム or カンバン？

### 3. Slack 設定

- このプロジェクトに関連する Slack チャンネルはありますか？
- チャンネル名を教えてください（複数可）

### 4. Confluence 設定

- Confluence を使っていますか？
- 使っている場合:
  - Confluence の URL
  - スペースキー（例: `BILLING`）

### 5. 関連リポジトリ

- このプロジェクトに関連する Git リポジトリはありますか？
- リポジトリ名と URL を教えてください（複数可）

### 6. 会議体

- 定例会議はありますか？
- 議事録の管理方法は？（Google Docs / 手動）

## 反映

ヒアリング結果を `project.yaml` に書き込んでください。未回答の項目はコメントアウトのまま残します。

完了したら `.envrc` に必要な認証情報の設定を案内してください。
